# 이 프로젝트가 뭐야? - 상세 설명

## 🎯 한 줄 요약
**GDGoC 성공회대학교 멤버들이 프로젝트 팀을 만들고 협업하기 위한 팀빌딩 플랫폼의 백엔드 서버입니다.**

## 🤔 왜 이런 걸 만들었나요?

GDGoC (Google Developer Groups on Campus) 동아리에서 활동하는 학생들이 프로젝트를 하고 싶을 때:
- "어떤 프로젝트를 할까?" 고민하는 사람들
- "팀원을 어떻게 구할까?" 걱정하는 사람들  
- "같이 할 사람 있나?" 찾는 사람들

이런 고민들을 해결하기 위해 만든 시스템입니다!

## 💡 이 시스템으로 뭘 할 수 있나요?

### 1. **아이디어 공유하기** 💡
- "이런 프로젝트 하면 어때?" 하는 아이디어를 올릴 수 있어요
- 다른 사람들의 아이디어도 구경할 수 있어요
- 마음에 드는 아이디어에 참여 신청을 할 수 있어요

### 2. **팀 구성하기** 👥
- 프론트엔드 개발자 필요해? ✅
- 백엔드 개발자 필요해? ✅  
- 디자이너 필요해? ✅
- 필요한 팀원을 모집하고 팀을 만들 수 있어요

### 3. **완성된 프로젝트 자랑하기** 🎨
- 프로젝트 완성하면 프로젝트 갤러리에 올려요
- 다른 팀들의 멋진 프로젝트도 구경할 수 있어요
- 포트폴리오로도 활용할 수 있어요

### 4. **커뮤니티 활동** 💬
- 프로젝트 모집 공고 올리기
- 다른 사람들과 소통하기

## 🔧 어떻게 만들어졌나요? (기술적인 부분)

### 백엔드 서버
- **Spring Boot**로 만들어진 자바 웹 서버입니다
- **REST API** 방식으로 프론트엔드와 통신해요

### 데이터베이스
- **MySQL**에 사용자 정보, 아이디어, 팀 정보 등을 저장해요
- **Redis**로 빠른 데이터 처리를 해요

### 보안
- **JWT 토큰**으로 로그인 상태를 관리해요
- **Spring Security**로 보안을 강화했어요
- **이메일 인증**으로 회원가입을 안전하게 해요

### API 문서
- **Swagger**를 사용해서 API 문서를 자동으로 만들어요
- 개발자들이 API를 쉽게 테스트할 수 있어요

## 📱 실제로 어떻게 작동하나요?

### 사용자 입장에서:
```
1. 회원가입 → 이메일 인증 → 관리자 승인
2. 로그인 후 아이디어 둘러보기
3. 마음에 드는 아이디어 찾기
4. 팀 참여 신청하기
5. 팀이 구성되면 프로젝트 시작!
6. 프로젝트 완성 후 갤러리에 등록
```

### 관리자 입장에서:
```
1. 회원 가입 승인/거부
2. 부적절한 아이디어나 게시글 관리
3. 프로젝트 갤러리 관리
4. 전체 활동 모니터링
```

## 🏗 프로젝트 구조 (간단하게)

```
백엔드 서버
├── 사용자 관리 시스템
│   ├── 회원가입/로그인
│   ├── 프로필 관리
│   └── 권한 관리
├── 아이디어 관리 시스템
│   ├── 아이디어 등록
│   ├── 아이디어 검색
│   └── 아이디어 지원
├── 팀 빌딩 시스템
│   ├── 팀 생성
│   ├── 팀원 모집
│   └── 팀 구성
├── 프로젝트 갤러리
│   ├── 완성 프로젝트 전시
│   └── 프로젝트 정보 관리
└── 관리자 시스템
    ├── 사용자 관리
    ├── 콘텐츠 관리
    └── 활동 관리
```

## 🎮 실제 화면 흐름 예시

### 일반 사용자
```
[로그인] 
   ↓
[메인 페이지] - 아이디어 목록 보기
   ↓
[아이디어 상세] - "이거 괜찮은데?"
   ↓
[팀 참여 신청] - "나도 참여하고 싶어요!"
   ↓
[승인 대기...]
   ↓
[팀 구성 완료!] - 프로젝트 시작
   ↓
[프로젝트 진행]
   ↓
[완성!] - 갤러리에 등록
```

### 아이디어 제안자
```
[아이디어 등록] - "이런 프로젝트 어때요?"
   ↓
[팀원 모집 설정]
   - 프론트엔드 2명
   - 백엔드 2명  
   - 디자이너 1명
   ↓
[지원자 확인]
   ↓
[팀원 선발]
   ↓
[팀 구성 완료] - 프로젝트 시작!
```

## 🔐 보안은 어떻게 되나요?

1. **이메일 인증**: 가짜 계정 방지
2. **JWT 토큰**: 안전한 로그인 유지
3. **관리자 승인**: 신뢰할 수 있는 멤버만 참여
4. **역할 기반 권한**: 일반 사용자와 관리자 권한 분리
5. **Spring Security**: 전문적인 보안 프레임워크 사용

## 📊 데이터는 어떻게 저장되나요?

### 사용자 데이터
- 이메일, 이름, 학번
- 역할(일반/관리자)
- 파트(프론트엔드/백엔드/디자인 등)
- 자기소개, SNS 링크 등

### 아이디어 데이터
- 제목, 주제, 설명
- 필요한 팀원 구성
- 상태(모집중/진행중/완료)
- 작성자 정보

### 팀 데이터
- 팀원 목록
- 각 팀원의 역할
- 프로젝트 진행 상태
- 프로젝트 일정

## 🚀 어떻게 실행하나요?

### 개발자라면:
1. 코드 다운로드 (git clone)
2. MySQL 데이터베이스 준비
3. `.env` 파일 설정 (데이터베이스 정보, 이메일 정보 등)
4. `./gradlew bootRun` 실행
5. `http://localhost:8080/swagger-ui.html` 접속해서 API 테스트

### 일반 사용자라면:
- 배포된 웹사이트나 앱을 사용하면 돼요!
- 백엔드 서버는 프론트엔드 개발자들이 사용하는 부분이에요

## 💻 누가 사용하나요?

- **GDGoC 성공회대학교 멤버들**: 프로젝트 팀 구성
- **관리자들**: 동아리 활동 관리
- **(나중에는) 다른 학교 GDGoC**: 비슷한 시스템 필요한 곳

## 🎓 이 프로젝트의 의미

1. **실전 경험**: 실제로 사용되는 서비스를 개발하는 경험
2. **협업 학습**: 팀으로 큰 프로젝트를 만드는 법 배우기
3. **기술 성장**: 최신 백엔드 기술 스택 활용
4. **포트폴리오**: 취업 준비에도 도움이 되는 프로젝트
